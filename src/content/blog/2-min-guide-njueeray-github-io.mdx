## 一张图理解这个项目

```
你（Ray / 访客）
     ↓
  判断力
     ↓
┌─────────────────────────────────────────┐
│          7 人 AI Agent 团队              │
│  Brain · PM · Dev · Researcher           │
│  Code Reviewer · Profile Designer · Brand│
└─────────────────────────────────────────┘
     ↓
  执行力
     ↓
这个站点 + OpenProfile 治理文档 + GitHub Discussions
```

Ray 是这支团队的判断者，不是执行者。Agent 们执行策略优先级最高的那一件事，Ray 决定哪些做、哪些不做、哪些调整。

**这不是「用 AI 写代码」——这是「人类判断力 × AI 执行力」的共生形态。**

## 这支团队在做什么

### 当前正在执行

这支团队遵循「Board 驱动 Ship 循环」：

```
每个工作会话：
  1. Recall  — 读取 Sprint Board，找到优先级最高的一件事
  2. Execute — 只做那一件事，做完
  3. Ship    — 更新 Board，触发对外发布（如有）
```

Sprint Board 是团队的唯一活跃状态源，每次会话都从它开始，以它结束。

### 已经完成的部分

| 阶段 | 交付物 |
|------|--------|
| GitHub Profile README | 动态 badge + 贡献蛇 + 3D 贡献图 |

## 这个项目的 3 个北极星

**1. 不是「AI 帮我做事」，是「人机共生进化」**

Agent 团队不是工具——它们是 Ray 认知系统的外化形态。Playbook 是协作认知的物理载体，会议纪要是决策记忆的外化存储。这支团队越运转，Ray 的判断力应该越增长——而不是退化。

**2. 过程即内容**

每一次版本决策、每一次会议争论、每一次选择「不做什么」——都作为博文发布。这不是事后总结，这是「正在做的时候就发」的 Build in Public 实践。

**3. 开源即示范**

[OpenProfile](https://github.com/njueeRay/OpenProfile) 仓库存放了这支团队的完整治理文档：Playbook、Agent 配置、设计决策、会议纪要。任何想用 AI 团队构建项目的人，都可以从这里复制一份出发点。
